## 1. Project Overview

### Vision
Linkr is a minimalist SaaS for saving, organizing, and rediscovering web bookmarks. Users paste a URL, Linkr automatically fetches the title, description, and favicon, then stores it in a searchable, tag-based library. Think of it as a private, fast alternative to browser bookmarks — with search, tags, and a public sharing option.

The target user is a developer, researcher, or knowledge worker who saves 10–50 links per week and currently loses them in browser folders or "read later" apps they never open again.

### Goals
- Users can save a URL in under 3 seconds (paste → save, no friction)
- Full-text search across titles, descriptions, and tags in under 200ms
- Simple free tier (up to 500 bookmarks) with a Pro plan (unlimited + collections)

### Success Metrics
- Time-to-first-bookmark under 60 seconds from sign-up
- 30-day retention above 40%
- Search query response time under 200ms at p95

## 4. Data Model

### Entities

**users**
| Field | Type | Notes |
|-------|------|-------|
| id | text (PK) | Clerk user ID (synced via webhook) |
| email | text | Unique |
| plan | enum('free','pro') | Default: 'free' |
| stripe_customer_id | text | Nullable |
| stripe_subscription_id | text | Nullable |
| created_at | timestamp | Default: now() |

**bookmarks**
| Field | Type | Notes |
|-------|------|-------|
| id | uuid (PK) | Generated |
| user_id | text (FK → users.id) | CASCADE delete |
| url | text | Not null |
| title | text | Fetched from URL metadata |
| description | text | Nullable |
| favicon_url | text | Nullable |
| is_public | boolean | Default: false |
| created_at | timestamp | Default: now() |
| search_vector | tsvector | Generated column for full-text search |

**tags**
| Field | Type | Notes |
|-------|------|-------|
| id | uuid (PK) | Generated |
| user_id | text (FK → users.id) | CASCADE delete |
| name | text | Lowercase, trimmed |
| color | text | Hex color for UI |

**bookmark_tags** (junction)
| Field | Type | Notes |
|-------|------|-------|
| bookmark_id | uuid (FK → bookmarks.id) | CASCADE delete |
| tag_id | uuid (FK → tags.id) | CASCADE delete |
| PRIMARY KEY | (bookmark_id, tag_id) | |

**collections** (Pro only)
| Field | Type | Notes |
|-------|------|-------|
| id | uuid (PK) | Generated |
| user_id | text (FK → users.id) | CASCADE delete |
| name | text | |
| is_public | boolean | Default: false |
| share_token | text | Unique, for public share links |

### Relationships
- One user → many bookmarks (1:N)
- One user → many tags (1:N)
- One bookmark ↔ many tags via bookmark_tags (M:N)
- One collection → many bookmarks (M:N via collection_bookmarks junction)

### Database Schema

```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  plan TEXT NOT NULL DEFAULT 'free' CHECK (plan IN ('free', 'pro')),
  stripe_customer_id TEXT,
  stripe_subscription_id TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE bookmarks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  url TEXT NOT NULL,
  title TEXT,
  description TEXT,
  favicon_url TEXT,
  is_public BOOLEAN NOT NULL DEFAULT FALSE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  search_vector TSVECTOR GENERATED ALWAYS AS (
    to_tsvector('english', COALESCE(title, '') || ' ' || COALESCE(description, '') || ' ' || url)
  ) STORED
);

CREATE INDEX idx_bookmarks_user_id ON bookmarks(user_id);
CREATE INDEX idx_bookmarks_search ON bookmarks USING GIN(search_vector);

CREATE TABLE tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  color TEXT NOT NULL DEFAULT '#6366f1',
  UNIQUE(user_id, name)
);

CREATE TABLE bookmark_tags (
  bookmark_id UUID REFERENCES bookmarks(id) ON DELETE CASCADE,
  tag_id UUID REFERENCES tags(id) ON DELETE CASCADE,
  PRIMARY KEY (bookmark_id, tag_id)
);

CREATE TABLE collections (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  is_public BOOLEAN NOT NULL DEFAULT FALSE,
  share_token TEXT UNIQUE DEFAULT encode(gen_random_bytes(16), 'hex'),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## 5. API Design

### Routes Overview
| Method | Path | Description | Auth |
|--------|------|-------------|------|
| GET | /api/bookmarks | List user's bookmarks (paginated, filterable) | Required |
| POST | /api/bookmarks | Create bookmark | Required |
| PATCH | /api/bookmarks/[id] | Update bookmark (title, tags, public) | Required (owner) |
| DELETE | /api/bookmarks/[id] | Delete bookmark | Required (owner) |
| POST | /api/bookmarks/fetch-meta | Fetch title/description from URL | Required |
| GET | /api/search | Full-text search across bookmarks | Required |
| POST | /api/webhooks/clerk | Sync Clerk user to database | Clerk signature |
| POST | /api/webhooks/stripe | Handle subscription events | Stripe signature |

### Key Endpoints Detail

**POST /api/bookmarks**
```json
// Request
{ "url": "https://example.com", "tagIds": ["uuid1", "uuid2"] }

// Response 201
{ "id": "uuid", "url": "...", "title": "Example", "favicon_url": "...", "tags": [...] }

// Errors
// 400 — invalid URL format
// 403 — free tier limit reached (500 bookmarks)
// 409 — bookmark already exists for this user
```

**GET /api/search?q=nextjs&tag=uuid&limit=20&offset=0**
```json
// Response 200
{
  "results": [{ "id": "...", "title": "...", "url": "...", "rank": 0.95 }],
  "total": 47
}
```

**POST /api/bookmarks/fetch-meta**
```json
// Request
{ "url": "https://example.com" }

// Response 200
{ "title": "Example Domain", "description": "...", "favicon_url": "https://..." }

// Timeout: 5s — if URL doesn't respond, return empty fields gracefully
```

### State Management
- **Server Components by default** — dashboard initial load fetches bookmarks server-side (no loading spinner on first render)
- **TanStack Query** — manages bookmark list, search results, tags with stale-while-revalidate caching
- **Optimistic updates** — delete and tag changes apply instantly, roll back on error
- **URL state** — search query and active tag stored in URL params (shareable, browser-back works)
- No global client state store needed — TanStack Query covers all server state

## 8. Authentication & Authorization

### Auth Flow
1. User visits landing page → clicks "Get Started"
2. Clerk sign-up modal (email magic link or Google OAuth)
3. On first sign-in, Clerk webhook fires → creates `users` record in database
4. Redirect to `/dashboard` (empty state with onboarding prompt)
5. Subsequent sign-ins: Clerk middleware validates session → app loads

### Protected Routes
- All `/(app)/*` routes — protected by `clerkMiddleware()` in `middleware.ts`
- Public routes: `/`, `/pricing`, `/bookmark/[id]` (if `is_public = true`)
- API routes: all `/api/*` except webhooks verify Clerk session via `auth()`

### Roles & Permissions
| Role | Can Do |
|------|--------|
| Free user | Save up to 500 bookmarks, full search, public sharing |
| Pro user | Unlimited bookmarks, collections, bulk export, CSV import |

Plan is stored in `users.plan` and checked server-side before gating Pro features.

### Session Management
- Clerk manages sessions via httpOnly cookies (no JWT handling needed)
- `auth()` server-side returns `userId` — use to scope all DB queries
- `useAuth()` client-side for conditional UI rendering
- Session refresh: automatic via Clerk SDK

## 13. Testing Strategy

### Unit Tests
Framework: Vitest. Test these modules in isolation:
- `src/lib/url-meta.ts` — mock fetch, test parsing of malformed HTML, timeouts, missing tags
- Plan gate logic — test free tier limit enforcement
- `src/lib/stripe/helpers.ts` — plan check utilities

### Integration Tests
- API route tests using `next/server` test utilities
- `POST /api/bookmarks` — test limit enforcement, duplicate detection, metadata fetch
- `GET /api/search` — test full-text ranking, empty queries, SQL injection resistance
- Webhook handlers — test with Stripe and Clerk test event payloads

### E2E Tests
Framework: Playwright. Critical flows to cover:
1. **Sign up → add first bookmark → search for it → find it** (core happy path)
2. **Free tier limit** — add 500 bookmarks, verify 501st is blocked with upgrade prompt
3. **Stripe upgrade** — click upgrade, complete Stripe test checkout, verify Pro features unlock
4. **Public bookmark** — mark bookmark public, copy link, open in incognito, verify accessible

Run: on every PR via GitHub Actions. Time budget: under 3 minutes.

### Data Flow
- Server Components fetch initial data directly via Drizzle (no API hop)
- Client mutations go through API routes (`/api/bookmarks`, `/api/tags`)
- TanStack Query manages client-side cache and optimistic updates
- All DB queries are scoped by `userId` from Clerk's `auth()` — never trust client-provided user IDs

### Key Patterns
- **Server Components by default.** Add `"use client"` only when component needs interactivity (forms, search bar)
- **All DB queries in `src/lib/db/`** — no inline SQL elsewhere
- **Plan gate pattern:** check `user.plan === 'pro'` server-side before returning gated data; never gate on client only
- **Webhook idempotency:** all webhook handlers check if event was already processed before mutating

## 16. Non-Negotiable Rules

1. **TypeScript strict mode.** No `any` types. `tsconfig.json` must have `"strict": true`.
2. **All API inputs validated with Zod.** Every POST/PATCH body goes through a Zod schema before touching the database.
3. **Webhook signatures verified.** Both Clerk and Stripe webhooks must verify the signature — reject unsigned requests with 401.
4. **All DB queries scoped by userId.** Never return data from a query that doesn't filter by the authenticated user's ID.
5. **No `.env.local` in git.** Verified by `.gitignore` and a pre-commit hook.
6. **Mobile-first responsive design.** All pages must work on 375px viewport. Test on mobile before marking any step complete.
7. **Graceful degradation on URL fetch.** If metadata fetch fails (timeout, blocked), save the bookmark with just the URL — never fail the save.
