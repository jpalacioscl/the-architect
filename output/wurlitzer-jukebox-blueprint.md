# Wurlitzer Jukebox — Blueprint

> Generated by The Architect on 2026-04-04
> Archetype: internal-tool (local personal web app)

---

## 1. Project Overview

### Vision
Una aplicación web local que transforma tu colección de MP3s en una rockola Wurlitzer interactiva. Corre en tu PC como un servidor local y es accesible desde cualquier dispositivo en la misma red LAN. El usuario ve una interfaz visual inspirada en la Wurlitzer 1015 "Bubbler" de los años 50 — con tubos de colores, efectos de neón y un carrusel de carátulas — y puede elegir álbumes y canciones como si estuviera frente a una rockola real.

### Goals
- Cargar la biblioteca musical escaneando automáticamente una carpeta configurada (estructura: `Artista/Album/cancion.mp3`)
- Mostrar carátulas de álbumes obtenidas automáticamente de la iTunes Search API
- Interfaz visual del Wurlitzer completamente interactiva: carrusel de álbumes, botones de selección de canciones, display animado
- Accesible desde cualquier dispositivo en la red LAN sin configuración adicional

### Success Metrics
- La app carga y muestra la biblioteca en menos de 2 segundos
- Cambiar de canción toma 1 clic — sin menús intermedios
- Se ve y se siente como una rockola real en pantalla completa

## 2. Tech Stack

| Capa | Tecnología | Por qué |
|------|-----------|---------|
| Servidor | Node.js + Express | Escanea el disco, sirve MP3s y hace proxy a la iTunes API — el browser no puede leer archivos locales directamente |
| Frontend framework | React 18 + Vite | La UI tiene suficiente estado interactivo (carrusel, audio, animaciones) para justificar React; Vite da hot reload instantáneo |
| Estilos | CSS puro (sin frameworks) | Un jukebox custom necesita CSS muy específico — gradientes, efectos neón, formas art deco. Tailwind estorba aquí |
| Fuentes | Google Fonts (Playfair Display + Orbitron) | Playfair para elegancia vintage, Orbitron para el display LCD del jukebox |
| Carátulas | iTunes Search API | Gratuita, sin API key, devuelve artwork de alta calidad buscando por artista + álbum |
| Audio | HTML5 Audio API nativo | Sin dependencias, soporta streaming, control total de play/pause/seek/volumen |
| Config | `config.json` | Un solo archivo donde el usuario pone la ruta de su carpeta de música |
| Package Manager | npm | Sin necesidad de pnpm para un proyecto de esta escala |

## 4. Data Model

No hay base de datos. Toda la información vive en memoria en el servidor y se reconstruye al iniciar.

### Estructura en memoria (JavaScript)

**Library** — devuelta por `/api/library`
```javascript
{
  artists: [
    {
      name: "Pink Floyd",
      albums: [
        {
          name: "The Wall",
          artist: "Pink Floyd",
          artworkUrl: "https://is1-ssl.mzstatic.com/...",  // de iTunes o cache local
          artworkCached: true,                              // si ya está en /cache/artwork/
          songs: [
            {
              title: "Another Brick in the Wall",
              filename: "01 - Another Brick in the Wall.mp3",
              track: 1,
              streamUrl: "/stream/Pink Floyd/The Wall/01 - Another Brick in the Wall.mp3"
            }
          ]
        }
      ]
    }
  ]
}
```

### Lógica del escáner (`scanner.js`)
1. Lee la carpeta raíz configurada en `config.json`
2. Cada subdirectorio de primer nivel = Artista
3. Cada subdirectorio de segundo nivel = Álbum
4. Cada `.mp3` dentro = Canción
5. Ordena canciones por nombre de archivo (prefijo numérico `01 -`, `02 -`, etc.)
6. Extrae el título limpio removiendo el prefijo numérico

## 5. API Design

### Routes Overview
| Método | Ruta | Descripción | Auth |
|--------|------|-------------|------|
| GET | `/api/library` | Árbol completo de artistas → álbumes → canciones | No |
| GET | `/api/artwork` | Carátula para un álbum. Params: `?artist=X&album=Y` | No |
| GET | `/stream/:artist/:album/:filename` | Streaming del archivo MP3 | No |
| GET | `/api/config` | Devuelve config actual (carpeta de música, versión) | No |

### Detalle de endpoints críticos

**GET /api/library**
```json
// Response 200
{
  "artists": [
    {
      "name": "Pink Floyd",
      "albums": [
        {
          "name": "The Wall",
          "artist": "Pink Floyd",
          "artworkUrl": "/cache/artwork/Pink_Floyd-The_Wall.jpg",
          "songs": [
            { "title": "Another Brick in the Wall", "track": 1, "streamUrl": "/stream/Pink Floyd/The Wall/01 - Another Brick in the Wall.mp3" }
          ]
        }
      ]
    }
  ]
}
// Error 500 si la carpeta de música no existe o no está configurada
```

**GET /api/artwork?artist=Pink+Floyd&album=The+Wall**
```
1. Verificar si existe /cache/artwork/Pink_Floyd-The_Wall.jpg
2. Si existe → redirigir al archivo cacheado
3. Si no → llamar a iTunes: https://itunes.apple.com/search?term=The+Wall+Pink+Floyd&entity=album&limit=1
4. Descargar la imagen de alta resolución (reemplazar "100x100" por "600x600" en la URL)
5. Guardar en /cache/artwork/
6. Devolver la imagen
// Fallback: si iTunes no encuentra nada → devolver imagen placeholder del jukebox
```

**GET /stream/:artist/:album/:filename**
```
- Construye la ruta: config.musicFolder / artist / album / filename
- Soporta Range headers (para seek en HTML5 Audio)
- Content-Type: audio/mpeg
- Error 404 si el archivo no existe
```

## 6. Frontend Architecture

### Páginas
Una sola página (`/`) — la rockola completa.

### Jerarquía de componentes

```
App
├── useLibrary() → carga /api/library al montar
├── useAudio()   → gestiona HTMLAudioElement global
│
└── Jukebox  (el frame visual completo)
    ├── BubbleTubes (izquierda)  — tubos animados decorativos
    ├── JukeboxTop               — domo superior
    │   ├── VinylDisc            — disco que gira con CSS animation mientras suena
    │   └── NowPlayingDisplay    — nombre artista / álbum / canción en estilo LCD
    ├── AlbumCarousel            — panel central
    │   ├── CarouselArrow left   — botón ← para rotar
    │   ├── AlbumCard[]          — carátulas (solo se ven 3: prev, active, next)
    │   └── CarouselArrow right  — botón → para rotar
    ├── SongSelector             — panel inferior de botones
    │   └── SongButton[]         — botones A1…A8 con título truncado
    ├── Controls                 — barra inferior
    │   ├── PlayPauseButton
    │   ├── PrevNextButtons
    │   ├── ProgressBar          — click para seek
    │   └── VolumeKnob           — estilo perilla rotativa con CSS
    └── BubbleTubes (derecha)
```

### Gestión de estado

Todo el estado vive en `App.jsx` y se pasa por props:
- `library` — árbol completo de música (cargado una sola vez)
- `selectedAlbum` — álbum activo en el carrusel
- `currentSong` — canción en reproducción
- `isPlaying` — boolean
- `currentTime / duration` — para la barra de progreso

`useAudio` encapsula el `HTMLAudioElement` y expone: `play()`, `pause()`, `seek(time)`, `setVolume(v)`, `loadSong(url)`.

No se necesita ningún gestor de estado externo (ni Redux, ni Zustand) — la app es lo suficientemente simple.

## 7. Design System

Inspiración visual: **Wurlitzer 1015 "Bubbler" (1946)** — la rockola más icónica de la historia. Colores cálidos, cromo dorado, tubos de burbujas de colores, neón ambarino.

### Colores
| Rol | Hex | Uso |
|-----|-----|-----|
| Fondo profundo | `#0F0800` | Fondo de página — negro cálido casi oscuro |
| Gabinete | `#2C1810` | Cuerpo principal del jukebox — caoba oscura |
| Cromo dorado | `#C9A84C` | Marcos, bordes, ornamentos metálicos |
| Cromo claro | `#E8D5A3` | Reflejos en el cromo, highlights |
| Ámbar neón | `#FF8C00` | Glow de botones activos, display iluminado |
| Rojo neón | `#FF1744` | Tubos de burbujas, indicador de reproducción |
| Azul neón | `#00B0FF` | Tubos de burbujas alternados |
| Crema | `#F5E6C8` | Texto sobre el gabinete oscuro |
| Texto muted | `#8B7355` | Labels secundarios, pistas inactivas |
| Vidrio domo | `rgba(255, 200, 50, 0.15)` | Overlay del domo superior con transparencia |

### Tipografía
| Rol | Fuente | Tamaño | Peso |
|-----|--------|--------|------|
| Título / marca | Playfair Display | 28px | 700 |
| Display LCD (Now Playing) | Orbitron | 14px / 12px | 400 |
| Botones canciones | Special Elite | 13px | 400 |
| Labels / controles | Playfair Display | 12px | 400 |

### Espaciado y Layout
- El jukebox ocupa exactamente el viewport (`100vh`) centrado — no hay scroll
- Ancho del jukebox: `min(500px, 95vw)` — se adapta a pantallas pequeñas
- Forma: el jukebox tiene un `border-radius` pronunciado en la parte superior (domo)
- Espaciado interno: base 8px

### Estilo de componentes
- **Sombras profundas**: `box-shadow: 0 0 40px rgba(201, 168, 76, 0.4)` para el glow dorado exterior
- **Neón activo**: `box-shadow: 0 0 10px #FF8C00, 0 0 20px #FF8C00, inset 0 0 5px rgba(255,140,0,0.3)` en botones activos
- **Vidrio del domo**: `backdrop-filter: blur(2px)` + gradiente radial ámbar
- **Burbujas**: `@keyframes bubble` — círculos SVG que suben verticalmente en loop
- **Vinilo girando**: `@keyframes spin` — rotación continua cuando `isPlaying === true`
- **Carrusel 3D**: transform `rotateY` con `perspective: 800px` — las tarjetas laterales aparecen en perspectiva

## 9. Orden de Construcción

**Paso 1: Scaffolding del proyecto**
```bash
mkdir wurlitzer && cd wurlitzer
npm init -y
npm install express cors
npm install -D vite @vitejs/plugin-react
mkdir -p server/routes client/src/{components,hooks,styles,utils} cache/artwork
```
Crear `config.json`:
```json
{ "musicFolder": "REEMPLAZAR_CON_TU_RUTA", "port": 3000 }
```

**Paso 2: Scanner de biblioteca (`server/scanner.js`)**
- Función `scanLibrary(musicFolder)` que lee el árbol de directorios
- Nivel 1 = Artistas, Nivel 2 = Álbumes, archivos `.mp3` = Canciones
- Ordena canciones por nombre de archivo
- Extrae título limpio: remover prefijo `01 - `, `02. `, etc. con regex
- Prueba manual: `node -e "const s = require('./server/scanner'); s.scanLibrary('./test-music').then(console.log)"`

**Paso 3: Ruta de artwork (`server/artwork.js` + `server/routes/artwork.js`)**
- Verificar caché en `cache/artwork/`
- Si no hay caché: fetch a `https://itunes.apple.com/search?term=${artist}+${album}&entity=album&limit=1`
- Parsear respuesta, obtener `artworkUrl100` y reemplazar `100x100bb` por `600x600bb`
- Descargar imagen con `https.get()` y guardar con `fs.createWriteStream()`
- Fallback: si iTunes falla, devolver `/client/src/assets/no-cover.png`
- Nombre del archivo cacheado: `Artist_Name-Album_Name.jpg` (sanitizado, sin caracteres especiales)

**Paso 4: Ruta de streaming (`server/routes/stream.js`)**
- Construir ruta: `path.join(config.musicFolder, artist, album, filename)`
- Soporte de `Range` headers para que HTML5 Audio pueda hacer seek:
```javascript
const range = req.headers.range
if (range) {
  const [start, end] = range.replace(/bytes=/, '').split('-').map(Number)
  // stream con fs.createReadStream({ start, end })
}
```
- `Content-Type: audio/mpeg`, `Accept-Ranges: bytes`

**Paso 5: Servidor Express (`server/index.js`)**
- Montar todas las rutas: `/api/library`, `/api/artwork`, `/stream`
- Servir el build del cliente desde `dist/` en producción
- En desarrollo: Vite corre en :5173, Express en :3000 — configurar proxy en `vite.config.js`
- **Crítico**: bind en `0.0.0.0` para acceso LAN:
```javascript
server.listen(config.port, '0.0.0.0', () => {
  const ip = getLocalIP() // usar módulo 'os' para obtener la IP de la interfaz LAN
  console.log(`Wurlitzer corriendo en:`)
  console.log(`  Local:   http://localhost:${config.port}`)
  console.log(`  Red LAN: http://${ip}:${config.port}`)
})
```

**Paso 6: Hook `useLibrary` y `useAudio`**
- `useLibrary`: `fetch('/api/library')` al montar, devuelve `{ library, loading, error }`
- `useAudio`: crea un `HTMLAudioElement` una sola vez con `useRef`, expone `{ play, pause, loadSong, seek, setVolume, currentTime, duration, isPlaying }`
- Usar `timeupdate` event para actualizar `currentTime` en el estado

**Paso 7: CSS variables y frame del jukebox (`global.css` + `jukebox.css`)**
- Definir todas las variables CSS del sistema de diseño en `:root`
- Construir la silueta del jukebox: un `div` con forma de jukebox usando `border-radius` asimétrico
  - Top: `border-radius: 120px 120px 0 0` para el domo
  - Laterales: bordes con `border` dorado y `box-shadow` con glow
- Gradientes de fondo del gabinete (madera oscura): `linear-gradient(180deg, #3d2419 0%, #2c1810 50%, #1a0f08 100%)`
- Grilla general: `display: grid` con zonas — top / carousel / selector / controls

**Paso 8: Componente `AlbumCarousel`**
- Estado: `activeIndex` (qué álbum está centrado)
- Renderizar siempre 3 tarjetas: `prev`, `active`, `next`
- La tarjeta activa: escala 1, opacidad 1, z-index alto
- Las laterales: `transform: rotateY(±40deg) translateX(±80px) scale(0.8)`, opacidad 0.6
- Transición: `transition: transform 0.4s ease, opacity 0.4s ease`
- Al hacer click en flecha: incrementar/decrementar `activeIndex` con wrap-around
- Cada `AlbumCard` muestra: imagen de carátula + nombre del álbum + nombre del artista

**Paso 9: Componente `SongSelector`**
- Recibe el álbum activo y lista sus canciones
- Renderizar botones estilo A1-A8 (máximo 8 canciones visibles — la rockola real tiene 8 por lado)
- Si el álbum tiene más de 8 canciones, mostrar solo las primeras 8
- Cada botón: número de pista + título truncado a 20 caracteres
- Botón activo (canción en reproducción): glow neón ambarino + clase `.active`
- Al click: llamar `loadSong(song.streamUrl)` + `play()`

**Paso 10: Componente `JukeboxTop` y vinilo animado**
- Display "Now Playing" con fondo oscuro y texto en Orbitron, simulando pantalla LCD
- Mostrar: título de canción (línea 1), artista — álbum (línea 2), tiempo transcurrido
- Vinilo: `div` circular con imagen de vinilo (`/assets/vinyl.png`) o CSS puro
- Animación: `@keyframes spin { from { rotate: 0deg } to { rotate: 360deg } }` — `animation-play-state: running/paused` según `isPlaying`

**Paso 11: Componente `BubbleTubes`**
- SVG o CSS puro con 3 tubos por lado
- Cada tubo: tubo vertical con círculos que suben con `@keyframes bubble`
- Colores alternando: rojo neón (#FF1744) y azul neón (#00B0FF)
- `animation-delay` diferente por burbuja para efecto orgánico

**Paso 12: Barra de controles (`Controls.jsx`)**
- Play/Pause: botón grande central
- Prev/Next: saltar a canción anterior/siguiente dentro del álbum activo
- Barra de progreso: `input[type=range]` estilizado como barra dorada
- Perilla de volumen: `input[type=range]` rotado 270° para parecer perilla
- Mostrar tiempo: `formatTime(currentTime) / formatTime(duration)`

**Paso 13: `npm run build` y modo producción**
```bash
npm run build  # Vite compila a dist/
node server/index.js  # Express sirve dist/ + API
```
Agregar script en `package.json`:
```json
{
  "scripts": {
    "dev": "concurrently \"node server/index.js\" \"vite client\"",
    "build": "vite build client --outDir ../dist",
    "start": "node server/index.js"
  }
}
```

**Paso 14: Polish y detalle final**
- Estado de carga: spinner ambarino mientras carga la biblioteca
- Estado vacío: si no hay música, mostrar mensaje con instrucciones para configurar `config.json`
- Error de archivo no encontrado: al fallar un stream, skip automático a la siguiente canción
- Auto-avance: cuando termina una canción (`audio.ended` event), play a la siguiente
- Fade out al cambiar de canción: bajar volumen a 0 en 300ms, cambiar canción, subir volumen

## 13. Estrategia de Testing

### Unit Tests
Framework: Vitest. Testear:
- `server/scanner.js` — con una carpeta de música de prueba falsa (fixtures), verificar que el árbol se construye correctamente y que los títulos se limpian bien
- `server/artwork.js` — mockear fetch a iTunes, verificar lógica de caché

### Tests manuales críticos (checklist antes de considerar terminado)
- [ ] La biblioteca carga correctamente desde la carpeta configurada
- [ ] Las carátulas aparecen (y se cachean — segunda carga es instantánea)
- [ ] El carrusel rota suavemente entre álbumes
- [ ] Click en botón de canción reproduce el audio
- [ ] El seek en la barra de progreso funciona
- [ ] El auto-avance al terminar una canción funciona
- [ ] Acceso desde otro PC en la LAN funciona
- [ ] La app se ve bien en pantalla completa y en tablet (horizontal)

## 16. Reglas No Negociables

1. **Servidor en `0.0.0.0`** — nunca en `localhost`. Sin esto, el acceso LAN no funciona.
2. **Range headers en streaming** — el endpoint `/stream` debe implementar soporte completo de `Range` para que el seek de HTML5 Audio funcione en todos los browsers.
3. **Sin frameworks de UI** — cero Tailwind, cero shadcn, cero Bootstrap. La autenticidad visual del jukebox depende de CSS completamente custom.
4. **Caché de carátulas obligatoria** — siempre guardar en `cache/artwork/` antes de devolver. Nunca hacer proxy directo a iTunes en cada request.
5. **Fallback visual siempre** — si falta una carátula, iTunes no responde, o un MP3 no existe: la app no puede romperse. Siempre mostrar un estado degradado graciosamente.
6. **Auto-avance implementado** — cuando termina una canción, la app avanza automáticamente a la siguiente. Esta es una funcionalidad core de cualquier jukebox.
